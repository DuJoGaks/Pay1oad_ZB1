[ Instant Images 워드프레스 플러그인 활용 문제 ]

📌 상황:
당신은 WordPress 사이트에 로그인할 수 있는 권한을 가지고 있습니다.  
해당 사이트는 'Instant Images'라는 플러그인을 사용하여 이미지를 삽입합니다.

📌 목표:
이미지 삽입 요청을 조작하여, 서버 내부에 숨겨진 `flag.txt` 파일에 접근하세요.

🧩 힌트:

1. 플러그인이 이미지를 가져오는 방식은 단순한 클라이언트 다운로드가 아닙니다.  
   서버가 직접 외부 URL에 요청을 보낸 뒤 이미지를 받아오는 구조입니다.

2. 요청이 실제로 서버에서 발생한다면,  
   외부 URL 대신 "내부 주소"를 요청하면 무슨 일이 벌어질까요?

3. 웹 브라우저로는 접근할 수 없는 특정 디렉토리가 존재합니다.  
   해당 위치에 플래그가 숨겨져 있습니다.

4. 요청을 보내는 방식을 확인해보세요.  
   (개발자 도구의 Network 탭, Burp Suite, Fiddler 등)

5. 응답이 이미지가 아닌 텍스트일 경우,  
   그 안에 **flag{...}** 형식의 문자열이 포함되어 있을 수 있습니다.
